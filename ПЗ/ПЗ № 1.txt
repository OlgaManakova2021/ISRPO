Необходимый теоретический материал:
PlantUML — это инструмент с открытым исходным кодом, позволяющий пользователям создавать диаграммы на простом текстовом языке. Помимо различных диаграмм UML, PlantUML поддерживает различные другие форматы, связанные с разработкой программного обеспечения, а также визуализацию файлов JSON и YAML  Графический вид создается автоматически.
PNG — растровый формат хранения графической информации
SVG — это вид графики, которую создают с помощью математического описания геометрических примитивов (линий, кругов, эллипсов, прямоугольников, кривых и так далее), которые и образуют все детали будущего изображения.
https://plantuml.com/ru/ - документация на русском.
https://plantuml.com/ru/skinparam - команда Skinparam (изменение цветов и шрифтов)

Пример диаграммы прецедентов (use case)

@startuml	//начало UML документа
:First Actor: as User_1	// объявление актора через псевдоним
actor Пользователь
(Закрыть окно) //объявление прецедента
(Открыть окно) as WO	//объявление прецедента через псевдоним
(Открыть файл) as (CO)

User_1 --> (WO) // назначение варианта использования
User_1 --> (CO)
Пользователь --> (WO)
User_1 --> (Закрыть окно)

left to right direction // изменение расположения блоков слева на право
@enduml	//окончание UML документа


Пример диаграммы деятельности

@startuml
:Вывалить мысли в текст;
if (Бред?) then (Возможно)
    :Дать посмотреть коллеге;
    if (Сойдет?) then (Да)
    else (Нет)
        :Помедитировать над текстом;
    endif
    :Разместить на //Хабре//;
else (Точно)
    stop
endif
@enduml


Порядок работы
    1. Ознакомится с теоретическим материалом.
    2. Составить UML диаграммы (с использованием  PlantUML). За основу взять  UML диаграммы построенные в курсовом проекте. Допускается, по согласованию с преподавателем, построить  UML диаграммы для дипломного проекта.
    3. Сохранять каждую UML диаграмму в отдельный в файл с расширением PNG. Имя UML диаграммы задавать по типу диаграммы (например, usecaseIvanov.png).
    4. Полученные файлы выгрузить в паку в репозитории на GitHub. 
    5. Результаты показать преподавателю.
